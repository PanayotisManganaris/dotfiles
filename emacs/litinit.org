*** GTD Philosophy
    clearly label actionables with some conveniences for tracking state changes over a project's history:

    keywords designed for use with hierarchically nested task-trees:
    - TODO any task to appear in the agenda. Any TODO heading with subheadings is a project. 
    - NEXT a priority task in an ongoing project -- a project with no NEXT is stuck (log time on)
      - also, NEXT is a catch-all for immediate actions: reply to, forward mail, return phone call, establish meeting, etc
    - WAIT any task requiring action from others before proceeding (log note on, time off)
    - DONE any completed task that stands only to be archived (log time on)
    - INACTIVE a project that is not prioritized -- cant be stuck (log note on)
    - CANCELED an aborted project that stands only to be archived (log note on, note off)			  

    hierarchically nested todos imply parents are dependent on children, sometimes siblings. So ORDERED property is assigned and enforced.
    overrule using NOBLOCKING property.
    
    captures documenting unplanned workaday occurrences as org trees can be balled with keywords too.
    - JOURNAL specific notes relevant to ongoing project. Journal entries should end up in reports (log time on)  
    - CONTACT add to professional address book and Contact/Customer Relationship Management (CRM) (log time on)
    - MEETING for any and all meetings, specific capture templates optional (log time on)                            
    perhaps category properties or tags are better -- but they lack the convenient time stamping

      "delegate for dired-open-remote-tmpfile"
      (copy-file file "/tmp/" 1)
      (let* ((extension (file-name-extension file))
	     (base (file-name-base file)))
	(format "/tmp/%s.%s" base extension)))
     
    (defun dired-open-remote-tmpfile (localprog)
      "copies file in remote dired to local tmp, opens with local program, and returns modified file to remote dired"
      (interactive "sLocal Program Name: ")
      (let* ((remote-file (dired-filename-at-point))
	     (tmpfile (copy-file-to-tmp remote-file)))
	(async-start-process localprog
			     localprog
			     (lambda (process-obj)
			       "move tmpfile back to remote directory"
			       (copy-file tmpfile remote-file 1))
			     tmpfile)))
